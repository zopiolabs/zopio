# Auth Policy

The `@repo/auth-policy` package provides a lightweight policy engine for making access control decisions in Zopio applications. It's designed to work seamlessly with Clerk authentication, using roles stored in Clerk's user metadata.

## Overview

The auth-policy package implements a simple policy-based authorization system that allows you to:

- Define policy rules as simple functions
- Check permissions based on user roles from Clerk
- Make context-aware authorization decisions
- Easily test with mock helpers

## Integration with Clerk

This package is designed to work with Clerk authentication. It uses the roles stored in Clerk's `publicMetadata.roles` field to make authorization decisions.

## Basic Usage

```tsx
import { can, createPolicyUser } from '@repo/auth-policy';
import { currentUser } from '@repo/auth/server';

// In a server component or API route
async function checkAccess() {
  const clerkUser = await currentUser();
  const policyUser = createPolicyUser(clerkUser);
  
  const allowed = await can({
    user: policyUser,
    action: 'plugin.install',
    resource: { id: 'plugin-123' }
  });
  
  if (allowed) {
    // Allow plugin installation
    return true;
  }
  
  return false;
}
```

## Policy Context

Each policy check requires a context object with the following properties:

```tsx
interface PolicyContext {
  user: {
    id: string;
    roles: string[];
    organizationId?: string;
    [key: string]: unknown;
  };
  action: string;
  resource?: unknown;
  context?: Record<string, unknown>;
}
```

## Available Policies

The following policies are currently implemented:

| Action | Description |
|--------|-------------|
| `plugin.install` | Controls who can install plugins |

## Adding New Policies

To add a new policy rule:

1. Create a new file in the `rules` directory
2. Implement your policy rule function
3. Register it in the `registry` object in `engine.ts`

```tsx
// rules/myFeature.ts
import type { PolicyRule } from '../types';

export const canAccessMyFeature: PolicyRule = ({ user }) => {
  return user.roles.includes('admin') || user.roles.includes('developer');
};

// Then in engine.ts
const registry: PolicyRegistry = {
  'plugin.install': canInstallPlugin,
  'myFeature.access': canAccessMyFeature
};
```

## Testing Helpers

The package includes helpers for testing:

```tsx
import { allowAll, denyAll } from '@repo/auth-policy/mock';

// Always allow access in tests
const result = await allowAll({ user: { id: '1', roles: [] }, action: 'any' });
// result is true

// Always deny access in tests
const result = await denyAll({ user: { id: '1', roles: [] }, action: 'any' });
// result is false
```
